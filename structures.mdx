---
title: 'Structures'
description: 'Understand better how Partoo data is structured'
---

Partoo data is organised around 5 main resources:


| Term             | Definition                                                                                                                                                                                                   |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Provider**     | A provider represents the entity that signs the contract with the client using Partoo solution & products. Can be Partoo itself or a reseller. A provider owns organizations, businesses, users, and groups. |
| **Organization** | An organization represents the legal entity, typically a commercial company, that owns businesses (or listings). It belongs to a provider.                                                                   |
| **Group**        | A group contains businesses. An organization can have several groups of businesses.                                                                                                                          |
| **Business**     | A business represents a listing. It belongs to an organization.                                                                                                                                              |
| **User**         | A user can be a Partoo app user or Partoo API user. A user belongs to an organization and has a role granting different levels of access on various resources within Partoo.                                 |
